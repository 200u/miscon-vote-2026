# ミス水戸一2026 投票システム

このプロジェクトは、完全な「端末保存型・QR集計方式」の投票システムです。

本番中、PCとiPadは接続しません。  
本番中、PCサーバー、ローカルネットワーク、インターネット接続は不要です。  
各iPadは単体で投票を受け付け、票をそのiPadのSafari内 `localStorage` に保存します。

投票終了後、各iPadの `admin.html` に表示されるQRコードまたは集計コードを、PCの `aggregate.html` に取り込んで合算します。`aggregate.html` はiPadと通信せず、QRまたは手入力された集計コードだけを読み取ります。

## ファイル構成

```text
public/
  index.html          iPad用入口
  vote.html           投票画面
  admin.html          iPadごとの管理画面
  result.html         そのiPad内だけの結果確認
  aggregate.html      PC用の総合集計画面
  shared.js           localStorage、投票、集計コード、CSV/TSV共通処理
  qrcode-v3.js        オフラインQRコード生成
  sw.js               オフラインキャッシュ
  manifest.webmanifest
  images/
    akane.jpg
    hamamero.jpg
    reina.jpg
    sorara.jpg
    yusepi.jpg
    ruta.jpg
    hina.jpg
```

`setup-server.js` は任意の初回読み込み補助用です。投票、保存、QR生成、集計には使いません。

## 静的サーバーについて

静的サーバーは、iPadへファイルを初回読み込みさせるための任意手段です。

- 本番中の投票には使いません。
- 本番中の保存には使いません。
- 本番中のQR生成には使いません。
- 本番中の集計には使いません。
- 一度iPadで読み込み、オフライン動作確認ができたら、本番中はPCサーバー不要です。
- PCとiPadを接続する必要はありません。

ファイルをiPadへ入れる方法は、学校や端末管理の環境に合わせて選んでください。例として、PWAキャッシュ、共有フォルダ、MDM、AirDrop、端末管理ツールなどがあります。

## 事前準備

1. 各iPadで投票システムを開く。
2. 必要ならホーム画面に追加する。
3. 各iPadで `admin.html` を開く。
4. 端末番号を `01` 〜 `07` に設定する。
5. 公演番号を `1` 〜 `4` から設定する。
6. `この公演を全リセット` を押す。
7. 必要に応じて `4公演すべて全リセット` を押す。
8. 機内モードにする。
9. 機内モードでも `vote.html` / `admin.html` / `result.html` が開けるか確認する。
10. 機内モードでも投票、保存、QR生成ができるか確認する。

## 本番開始前

各iPadで以下を確認します。

- 端末番号が重複していない
- 公演番号が正しい
- 本番対象の公演が全リセット済み
- 投票開始が押されている
- `vote.html` が表示されている

観客に渡す前に、管理画面ではなく投票画面を表示してください。

## 投票中

投票中は以下の状態で運用できます。

- PC不要
- Wi-Fi不要
- サーバー不要
- インターネット不要
- 各iPad単体で投票受付
- 票は各iPadの `localStorage` に保存

Safariの履歴・Webサイトデータを削除すると、そのiPad内の票は消えます。プライベートブラウズでは使わないでください。

## 投票終了後

1. 各iPadで `admin.html` を開く。
2. 公演番号が正しいことを確認する。
3. `投票締切` を押す。
4. QRコードと集計コードを表示する。
5. QRが読めない場合に備えて、集計コード文字列も確認する。

集計コード形式:

```text
M26/公演番号/端末番号/あかね-はまめろ-れいな-そらら-ゆせぴ-るた-ひな/チェックサム
```

例:

```text
M26/1/01/12-9-6-10-14-5-8/AB
```

チェックサムが一致しないコードは取り込めません。

## 集計

PCで `aggregate.html` を開きます。PCはiPadと通信しません。

1. 各iPadのQRコードを読み取る。
2. QRが読めない場合は集計コードを手入力する。
3. 同じ公演番号・同じ端末番号を再取り込みした場合は上書きされる。
4. 公演ごとに `01` 〜 `07` が取り込まれたか確認する。
5. 未取り込み端末が表示された場合は、そのiPadのQRまたはコードを取り込む。
6. 公演別結果と4公演合算結果を確認する。
7. Googleスプレッドシート用TSVをコピーする。
8. 必要ならCSVを出力する。

`aggregate.html` でできること:

- QR読み取り
- 集計コード手入力
- チェックサム検証
- 公演番号 1〜4 の検証
- 端末番号 01〜07 以外の警告
- 同じ公演・同じ端末の上書き
- 公演別結果表示
- 4公演合算結果表示
- 未取り込みiPad表示
- 公演別TSVコピー
- 4公演まとめTSVコピー
- 総合結果TSVコピー
- CSV出力
- 総合集計データ削除

## 正しいテスト手順

1. 各iPadで投票システムを開く。
2. `admin.html` で端末番号と公演番号を設定する。
3. 本番対象の公演を全リセットする。
4. 投票開始を押す。
5. `vote.html` で数票テスト投票する。
6. `admin.html` で票数が増えているか確認する。
7. Safariを閉じて再度開き、票が残っているか確認する。
8. iPadを機内モードにする。
9. 機内モードで `vote.html` / `admin.html` が開くか確認する。
10. 機内モードで投票、保存、QR生成ができるか確認する。
11. PCの `aggregate.html` でQRまたは集計コードを取り込む。
12. 取り込み結果が正しく合算されるか確認する。
13. 最後に各iPadで全リセットし、本番票が0から始まる状態にする。

## 注意

- 票は各iPad内にしかありません。
- 端末を紛失、初期化、またはSafariデータ削除した場合、その端末の票は復元できません。
- 端末番号を重複させると、総合集計では同じ公演・同じ端末番号のデータが上書きされます。
- 公演番号を間違えると、別公演の票として取り込まれます。
